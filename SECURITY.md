# Security Policy 🛡️

Project Claudia is a security-first platform dedicated to ensuring the integrity of AI-generated code. We take security vulnerabilities very seriously and appreciate the community's help in identifying and resolving them.

## Supported Versions

Currently, the following versions of Project Claudia are being supported with security updates:

| Version | Supported |
| :------ | :-------- |
| 1.x     | :white_check_mark: |
| < 1.0   | :x: |

## Reporting a Vulnerability

As an adversarial security engine, we strive for zero-day resilience. If you discover a security vulnerability within Project Claudia, please do not use the public issue tracker. Instead, follow these steps:

1. **Private Reporting**: Use [GitHub's Private Vulnerability Reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-post-reporting/about-reporting-a-vulnerability) feature to report the issue directly to the maintainers.
2. **Details**: Provide a detailed description of the vulnerability, including steps to reproduce, potential impact, and any suggested fixes.
3. **Response**: You can expect an initial acknowledgement within 48 hours. We will provide updates as we work toward a resolution.

We believe in **Coordinated Vulnerability Disclosure** and will provide credit in our security advisories for validated findings.
