# Getting Support for Project Claudia 🛡️

Thank you for using Project Claudia! Whether you're an enterprise security engineer or an open-source contributor, we're here to help you navigate the adversarial AI landscape.

## 💬 Community Support (Recommended)
For general questions, architecture discussions, and "How-to" guides, please use our GitHub-native channels. This ensures that the community can benefit from the discussion and that our knowledge base remains public and searchable.

- **[GitHub Discussions](https://github.com/ansonsaju/project-claudia/discussions)**: Best for brainstorming, architectural advice, and showcasing your implementations.
- **[GitHub Issues](https://github.com/ansonsaju/project-claudia/issues)**: Best for reporting bugs or requesting specific technical features.

## 🐞 Reporting Bugs
If you've found a bug, please check the [Issue Tracker](https://github.com/ansonsaju/project-claudia/issues) first to see if it's already been reported. If not, use the **Bug Report** template to provide as much context as possible.

## 🔐 Security Vulnerabilities
If you've discovered a security vulnerability, **do not open a public issue.** Please follow the instructions in our **[Security Policy](https://github.com/ansonsaju/project-claudia/blob/main/SECURITY.md)** to report it privately.

## 🚀 Enterprise Assistance
For organizations needing deep integration support, technical audits, or custom agent development, please reach out to the lead maintainer via their professional portfolio or LinkedIn listed in the [README](https://github.com/ansonsaju/project-claudia/blob/main/README.md).
